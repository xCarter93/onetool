    **Fix 1 — UPLD-02: BOM stripping in transform-csv.ts:**

    In `parseCsvData()`, strip UTF-8 BOM before passing to PapaParse:
    ```typescript
    const cleanContent = fileContent.charCodeAt(0) === 0xFEFF
      ? fileContent.slice(1)
      : fileContent;
    ```
    Pass `cleanContent` (not `fileContent`) to `Papa.parse()`.

    **Fix 2 — UPLD-03: dynamicTyping in transform-csv.ts:**

    In `parseCsvData()`, change `dynamicTyping: true` to `dynamicTyping: false`. This preserves phone numbers and zip codes as strings. The existing `transformValue()` function already handles type coercion based on schema field data types, so downstream code is unaffected.

    **Fix 3 — MAP-04: Headers-only AI request in use-import-wizard.ts:**

    In `handleFileSelect`, after parsing the CSV with `parseCsvData(content)`:
    1. Extract headers: `const headers = rows.length > 0 ? Object.keys(rows[0]) : [];`
    2. Extract sample rows (first 5): `const sampleRows = rows.slice(0, 5);`
    3. Send `{ headers, sampleRows, entityType }` to `/api/analyze-csv` instead of `{ csvContent, entityType }`.

    The full CSV content stays in the hook state for later use during import — only the subset goes to the AI.

    **Fix 4 — MAP-05: Auth + maxDuration on analyze-csv/route.ts:**

    Add at top of file:
    ```typescript
    import { auth } from "@clerk/nextjs/server";
    export const maxDuration = 60;
    ```

    At start of POST handler, before any processing:
    ```typescript
    const { userId } = await auth();
    if (!userId) {
      return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
    }
    ```

    Also update the route to accept `{ headers, sampleRows, entityType }` instead of `{ csvContent, entityType }`:
    - Destructure: `const { headers, sampleRows, entityType } = body;`
    - Build CSV sample for the prompt: join headers + sample rows into a CSV string
    - Remove the `csvContent` validation and MAX_FILE_SIZE check (no longer receiving full content)
    - Keep entityType validation
    - Update the prompt to use the CSV sample string instead of `csvContent`

    This is the established auth pattern — all 6 existing API routes in the project use the same `auth()` approach (confirmed in stripe-connect routes).
