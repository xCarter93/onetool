  <action>
Rewrite the POST handler in `apps/web/src/app/api/analyze-csv/route.ts` to call mapSchemaTool and validateDataTool directly instead of using the Mastra agent loop. Specifically:

1. **Remove** the `import { mastra } from "@/mastra"` import and all agent-related code (agent.generate, toolResults extraction, the CSV reconstruction block on lines 56-69, and the ToolResult type).

2. **Import** the tool execute functions directly:
   ```typescript
   import { mapSchemaTool } from "@/mastra/tools/map-schema-tool";
   import { validateDataTool } from "@/mastra/tools/validate-data-tool";
   ```

3. **Call mapSchemaTool.execute() directly** with the headers and sampleRows already provided by the frontend:
   ```typescript
   const mapResult = await mapSchemaTool.execute({
     context: {
       entityType: entityType || "clients",
       headers,
       sampleRows,
     },
   });
   ```
   Note: Mastra createTool's execute receives input via `context` property in v1 beta. Verify this by checking the mapSchemaTool source — the execute function destructures `{ entityType, headers, sampleRows }` from its input parameter. If Mastra wraps it, adapt accordingly (may be `.execute({ entityType, headers, sampleRows })` directly — test both patterns).

4. **Call validateDataTool.execute() directly** with the mapResult:
   ```typescript
   const validateResult = await validateDataTool.execute({
     context: {
       entityType: entityType || "clients",
       mappings: mapResult.mappings,
       sampleRows,
     },
   });
   ```

5. **Build the CsvAnalysisResult** from the direct tool results. Calculate a real overall confidence score as the average of all mapping confidences (not hardcoded 0.8):
   ```typescript
   const avgConfidence = mapResult.mappings.length > 0
     ? mapResult.mappings.reduce((sum, m) => sum + m.confidence, 0) / mapResult.mappings.length
     : 0;

   const analysisResult: CsvAnalysisResult = {
     entityType: entityType || "clients",
     detectedFields: mapResult.mappings,
     validation: {
       isValid: validateResult.isValid,
       errors: validateResult.errors,
       warnings: validateResult.warnings,
       missingRequiredFields: validateResult.missingRequiredFields,
     },
     suggestedDefaults: validateResult.suggestedDefaults,
     confidence: Math.round(avgConfidence * 100) / 100,
     sampleData: sampleRows,
   };
   ```

6. **Keep** the existing auth guard, input validation, maxDuration=60, and error handling. Only the agent orchestration changes.

7. **Do NOT delete** the Mastra agent file or tools — the agent may still be used for other purposes. Only change how this route calls the tools.

**Why this works:** The mapSchemaTool and validateDataTool contain purely deterministic logic (string matching, schema lookup, validation). No LLM is needed. Calling them directly eliminates 4+ GPT-4o round-trips and reduces the endpoint from 15-90+ seconds to under 100ms.

  <name>Task 2: Add AbortController with 30s timeout on frontend fetch</name>
  <files>apps/web/src/app/(workspace)/clients/import/hooks/use-import-wizard.ts</files>
  <action>
In `use-import-wizard.ts`, update the `handleFileSelect` callback to add an AbortController with a 30-second timeout on the fetch call to `/api/analyze-csv`:

1. **Create AbortController and timeout** before the fetch:
   ```typescript
   const controller = new AbortController();
   const timeoutId = setTimeout(() => controller.abort(), 30_000);
   ```

2. **Pass signal to fetch:**
   ```typescript
   const response = await fetch("/api/analyze-csv", {
     method: "POST",
     headers: { "Content-Type": "application/json" },
     body: JSON.stringify({ headers, sampleRows, entityType: "clients" }),
     signal: controller.signal,
   });
   ```

3. **Clear timeout on success** (after the fetch resolves):
   ```typescript
   clearTimeout(timeoutId);
   ```

4. **Handle AbortError specifically** in the catch block. Add a check before the existing error handling:
   ```typescript
   } catch (err) {
     clearTimeout(timeoutId);
     const isTimeout = err instanceof DOMException && err.name === "AbortError";
     console.error("Error analyzing CSV:", err);
     toast.error(
       isTimeout ? "Analysis Timed Out" : "Analysis Failed",
       isTimeout
         ? "The analysis took too long. Please try again with a smaller file."
         : err instanceof Error
           ? err.message
           : "Failed to analyze CSV file"
     );
     setState((prev) => ({ ...prev, isAnalyzing: false }));
   }
   ```

5. **Keep everything else unchanged** — the rest of handleFileSelect, all other handlers, and the hook's return value stay the same.
