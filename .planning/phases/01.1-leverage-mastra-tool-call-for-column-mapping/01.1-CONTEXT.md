# Phase 01.1: Leverage Mastra Tool Call for Column Mapping - Context

**Gathered:** 2026-03-15
**Status:** Ready for planning

<domain>
## Phase Boundary

Replace the deterministic synonym/substring column mapping in `mapSchemaTool` with full LLM-powered mapping via Mastra structured output. The tool's external interface stays the same — only the internal matching logic changes from hardcoded rules to an LLM call using GPT-5 nano.

</domain>

<decisions>
## Implementation Decisions

### LLM integration approach
- Replace the internals of `mapSchemaTool.execute()` — swap synonym map + substring matching for an LLM call
- Keep the same tool interface (input: headers, sampleRows, entityType → output: mappings, unmappedColumns, missingRequiredFields)
- API route (`analyze-csv/route.ts`) stays unchanged — still calls `mapSchemaTool.execute()` directly
- Model: GPT-5 nano (cheapest, fastest — structured extraction task doesn't need larger model)
- Use Mastra's structured output via `generate()` with a Zod schema for the response

### Prompt & output contract
- LLM receives: CSV headers, 3-5 sample rows, and full CLIENT_SCHEMA_FIELDS definition (field names, types, required/optional, enum options, groups)
- LLM returns structured output matching current FieldMapping interface: csvColumn, schemaField, confidence (0-1), dataType, isRequired, sampleValue
- LLM also returns unmapped columns list (columns it couldn't match to any schema field)
- Confidence scores are real per-field scores from the LLM (replaces hardcoded 0.8/0.9/1.0)
- Low-confidence mappings (<0.7) shown as "suggested" in UI; high-confidence auto-accepted
- No reasoning/explanation in output — just mappings + confidence scores

### Fallback & error handling
- On LLM failure (timeout, API error, rate limit): show error and let user map manually — all columns shown as unmapped with "Do not import" default
- No deterministic fallback — LLM is the sole mapping strategy
- Zod validation on LLM response — if structured output fails schema validation, treat same as API error (manual mapping)
- 15-second timeout on the LLM call inside the tool, separate from API route's maxDuration
- Delete all deterministic matching code (synonym map, substring logic, HEADER_SYNONYMS) — clean removal

### Claude's Discretion
- Whether to use agent.generate() with the tool or call LLM directly inside tool execute() — pick cleanest approach given existing code
- Exact prompt wording and structure
- How to format CLIENT_SCHEMA_FIELDS in the prompt (JSON, markdown table, etc.)
- Zod schema design for the structured output

</decisions>

<specifics>
## Specific Ideas

- GPT-5 nano chosen specifically because column mapping is a structured extraction task — doesn't need reasoning power of larger models
- The tool interface must not change — this is a drop-in replacement of internals only
- User explicitly chose no deterministic fallback: if AI fails, users map manually via the existing dropdown UI

</specifics>
