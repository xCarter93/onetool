1. **Add imports** at top:
   ```
   import { generateObject } from "ai";
   import { openai } from "@ai-sdk/openai";
   ```

2. **Create LLM response Zod schema** (separate from the tool's outputSchema):
   - Use `z.nullable()` NOT `z.optional()` for OpenAI structured output compatibility
   - Schema: `{ mappings: [{ csvColumn, schemaField, confidence, sampleValue: z.string().nullable() }], unmappedColumns: [string] }`

3. **Create `buildMappingPrompt` function:**
   - Takes entityType, headers, sampleRows, schema
   - Formats schema fields as JSON array with fieldName, type, required, group, and allowedValues (if enum)
   - Includes CSV headers as JSON array
   - Includes first 5 sample rows as JSON
   - Instructs LLM to use ONLY exact field names from schema
   - Explicitly explains dot-namespace convention: "Fields like contact.firstName use dot-notation as a namespace separator, not nested objects. Use the exact string including the dot."
   - Instructs: "Each schema field should be mapped to at most one CSV column"
   - Instructs: "Provide confidence 0.0-1.0 based on how well the header and sample data match the field"

4. **Create `postProcessMappings` function:**
   - Validates each mapping's schemaField exists in the schema (reject invalid ones to unmappedColumns)
   - Sorts by confidence descending, keeps highest-confidence mapping when duplicates exist for same schemaField
   - Enriches each mapping with `dataType` and `isRequired` from the schema
   - Converts nullable sampleValue to optional (undefined)
   - Ensures all headers not in mappings are in unmappedColumns
   - Computes missingRequiredFields (required schema fields not mapped)

5. **Rewrite `execute` function:**
   - Delete ALL deterministic code: HEADER_SYNONYMS map, synonym lookup, substring matching, special case mappings, the entire forEach loop
   - Call `generateObject({ model: openai("gpt-5-nano"), schema: llmMappingSchema, prompt: buildMappingPrompt(...), abortSignal: AbortSignal.timeout(15_000) })`
   - Pass result to `postProcessMappings`
   - Wrap in try/catch: on ANY error, `console.error` the error and return `{ mappings: [], unmappedColumns: [...headers], missingRequiredFields: [required fields from schema] }`

6. **Keep the tool's inputSchema and outputSchema unchanged** — same Zod schemas, same field names

7. **Update route comment** in `apps/web/src/app/api/analyze-csv/route.ts`:
