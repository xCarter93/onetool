    3. In `handleFileSelect` success path: After setting state with analysisResult, add auto-advance:
       ```
       setAnalysisError(null);
       setManualOverrides(new Set()); // Reset on new file
       setTimeout(() => { navigateTo("map"); }, 1000);
       ```
       Per research pitfall #1: the 1s delay is sufficient for React to commit state before the step guard checks.

    4. In `handleFileSelect` catch block: Instead of just toast, also set analysisError:
       ```
       setAnalysisError(isTimeout ? "The analysis took too long..." : err instanceof Error ? err.message : "Failed to analyze CSV file");
       ```
       Keep the toast as well for immediate feedback.

    5. In `handleMappingChange`: Mark column as manually overridden:
       ```
       setManualOverrides(prev => new Set(prev).add(csvColumn));
       ```

    6. In `startOver`: Reset manualOverrides and analysisError:
       ```
       setManualOverrides(new Set());
       setAnalysisError(null);
       ```

    7. Add `handleRetryAnalysis` callback: Re-triggers analysis on the current file. If `state.file` and `state.fileContent` exist, call `handleFileSelect(state.file, state.fileContent)`.

    8. Add `handleClearFile` callback: Resets file-related state (file, fileContent, analysisResult, mappings, analysisError) and stays on upload step.

    9. Add `handleProceedUnmapped` callback: Creates mappings array with all columns mapped to `__skip__` (confidence 0), clears analysisError, navigates to map step. This lets user manually map from scratch.

    10. Return new values: `manualOverrides`, `analysisError`, `handleRetryAnalysis`, `handleClearFile`, `handleProceedUnmapped`.

       ```

       ```
